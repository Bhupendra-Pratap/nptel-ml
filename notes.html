WEEK 1

ML Fundamentals & Probability

10 questions
🗂️ Types of Learning ▾
SupervisedLabelled training data. Learns a mapping input → output. Includes classification (discrete output) and regression (continuous output).
UnsupervisedNo labels. Finds hidden structure — clustering, density estimation. e.g. grouping documents, social-network communities.
ReinforcementAgent learns by interacting with an environment, receiving reward signals. e.g. game-playing agents.
Exam trick: "Grouping" / "clustering" with no labels = unsupervised. Predicting a numeric value = regression. Predicting a category = classification.
📊 Classification vs Regression ▾
ClassificationOutput is a discrete class label. e.g. spam/not-spam, cricket player type, traffic jam yes/no.
RegressionOutput is a real number. e.g. monthly sales in ₹, confirmation probability of a train ticket (0–1 fraction), temperature in °C.
Careful: "Probability" outputs are still regression (continuous [0,1]). "Predicting if a user likes a song" — binary, so classification.
🏷️ Feature Types ▾
NumericalContinuous or discrete counts: height, age, weekly expenditure, study hours.
CategoricalFinite set of unordered labels: branch of engineering, ethnicity, blood group.
Key rule: Number of legs is numerical (counts); engineering branch is categorical (no natural order).
⚖️ Bias–Variance Tradeoff ▾
BiasError from wrong assumptions (underfitting). High bias = model too simple.
VarianceSensitivity to training data fluctuations (overfitting). High variance = model too complex.
As model complexity ↑ Bias ↓  |  Variance ↑
Underfitting → High Bias, Low Variance
Overfitting → Low Bias, High Variance
🎲 Probability Essentials ▾
Addition rule P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
Complement P(neither A nor B) = 1 − P(A ∪ B)
Independent events P(A ∩ B) = P(A) · P(B)
Eigenvalues from trace/det: trace = λ₁+λ₂, det = λ₁·λ₂. Solve the quadratic to find eigenvalues.
WEEK 2

Linear Regression & Feature Selection

10 questions
📉 Linear Regression Basics ▾

Finds the best-fit line/hyperplane by minimizing the Residual Sum of Squares (RSS):

RSS = Σᵢ [yᵢ − f(xᵢ)]²
Key insight: A high negative coefficient does NOT automatically mean the feature is unimportant. Coefficient magnitude depends on feature scale. Never drop a feature based on coefficient sign alone — need standardization first.
Residuals: Lower is always better. Residual = actual − predicted.
🔒 Regularization: Ridge vs Lasso ▾
PropertyRidge (L2)Lasso (L1)
PenaltyΣβᵢ²Σ|βᵢ|
Effect on coeff.Shrinks toward 0Can set exactly to 0
Feature selection❌ No✅ Yes
As λ increasesCoefficients shrinkCoefficients shrink / zero out
For feature selection → always choose Lasso. Ridge keeps all features; Lasso performs automatic selection by zeroing coefficients.
🔍 Feature Selection Methods ▾
Best SubsetTries all 2ᵖ subsets of p features. Very expensive — O(2ᵖ) model fits.
Forward StepwiseStart with no features, add best one at a time. Works when p > n (more features than samples).
Backward StepwiseStart with all features, remove worst one at a time. Requires n > p to begin.
Exam rule: If features > data points → use Forward Stepwise (Backward can't start).
📍 K-Nearest Neighbours (KNN) Regression ▾

For a query point, find the K nearest training points (Euclidean distance) and predict the average of their output values.

Euclidean distance d(x, q) = √[(x₁-q₁)² + (x₂-q₂)² + (x₃-q₃)²]
Steps for exam: (1) Compute distance to all training points. (2) Pick K smallest. (3) Average their y-values.
WEEK 3

Logistic Regression & Discriminant Analysis

9 questions
📈 Logistic Regression ▾
Logistic (sigmoid) function σ(z) = 1 / (1 + e^(−z)) where z = w·x
σ(z) itselfNon-linear in the weights (sigmoid shape).
Decision boundaryLinear in the weights (hyperplane where z = 0).
y|x distributionBernoulli distribution (binary output).
Outlier resistanceMore robust than LDA/QDA. The sigmoid saturates, reducing outlier influence.
🔵 LDA vs PCA for Dimensionality Reduction ▾
LDAPCA
TypeSupervisedUnsupervised
GoalMaximize class separabilityMaximize variance
Use whenClasses are well-separated in label spaceNo class labels available
Exam rule: If data shows distinct class clusters → LDA. If no labels / pure compression → PCA.
📐 LDA Decision Boundary Calculation ▾

For two Gaussian classes with equal covariance (identity matrix), LDA boundary passes through the midpoint of the means and is perpendicular to the line joining them.

Means: μ₁ = (−1, 2), μ₂ = (1, 4) → midpoint = (0, 3) Direction vector: (1,4)−(−1,2) = (2,2) → normal to boundary
Boundary equation: x + y = 3
Non-linear to linear trick: Y = mX² + c can be linearized by substituting X' = X². Then apply linear regression on (X', Y).
WEEK 4

Perceptron & SVM

6 questions
🧠 Perceptron Algorithm ▾

A simple linear binary classifier. It converges only if the data is linearly separable.

Critical exam point: If data is NOT linearly separable → perceptron will NEVER achieve zero training error. It does not converge.
Outputŷ = sign(w·x + b)
Update ruleOnly updates when a mistake is made: w ← w + y·x
🎯 SVM vs Perceptron ▾
PerceptronFinds any separating hyperplane. Not unique — depends on initialization and data order.
Linear SVMFinds the maximum-margin hyperplane. Unique, optimal solution via convex optimization.
They will NOT always produce the same classifier. SVM selects the unique maximum-margin solution; perceptron can land on many different hyperplanes.
🔧 Regularization in Practice ▾
L2 (Ridge) λ ↑Coefficients shrink toward 0 (but never exactly 0).
SVM C paramC = 1/λ. Higher C = less regularization = smaller margin = fits training data more tightly.
RBF kernel γHigher γ = tighter decision boundary around each point.
WEEK 5

Neural Networks & MLE

9 questions
🔢 Counting Neural Network Parameters ▾
Parameters = weights + biases Weights: (inputs × hidden) + (hidden × outputs)
Biases: one per neuron in hidden + output layers

Example: 3 inputs → 1 hidden layer (3 neurons) → 1 output

Weights: 3×3 + 3×1 = 9 + 3 = 12
Biases: 3 (hidden) + 1 (output) = 4
Total = 16 parameters
📊 Maximum Likelihood Estimation (MLE) ▾

MLE finds the parameter value that makes the observed data most probable.

Normal distribution — MLE of mean μ_MLE = (1/n) Σ xᵢ (sample mean)
For Normal distribution: MLE of μ = sample mean. For data {30, 50, 69, 2, 99}: mean = 250/5 = 50.
🔙 Backpropagation & Sigmoid ▾
Sigmoid and its derivative σ(x) = 1/(1+e^−x)
σ'(x) = σ(x)·(1 − σ(x))
BackpropChain rule applied backward through the network. Gradient ∂L/∂w = ∂L/∂ŷ · ∂ŷ/∂w.
Weight updatew ← w − η · ∂L/∂w where η is learning rate.
ANN vs SVM: ANN loss surface = multiple local minima (non-convex). SVM loss = single global minimum (convex). ANN can give different results each run due to random init.
🧱 Neural Network Representational Power ▾
1 hidden layerCan represent any Boolean function (with enough hidden units).
2 hidden layersCan approximate any continuous function within any tolerance.
OverfittingMore hidden nodes / layers → MORE overfitting risk, not less.
WEEK 6

Decision Trees

8 questions
🌳 Decision Tree Structure ▾
Non-leaf nodeSpecifies a test condition on a feature (e.g. "age < 30?").
Leaf nodeSpecifies the predicted class label (or value).
Multi-classDecision trees and KNN work natively for multiple classes — no modification needed.
📐 Impurity Measures (Splitting Criterion) ▾
Cross-entropy (information gain) H = −Σ pᵢ log₂(pᵢ)
Gini impurity Gini = 1 − Σ pᵢ²

Select the attribute that results in the lowest weighted impurity after the split (highest information gain).

Root selection: The attribute with highest information gain (lowest post-split entropy) becomes the root.
✂️ Pruning & Bias-Variance in Trees ▾
Deeper treeBias ↓, Variance ↑ (overfits). Memorizes training data.
Shallow treeBias ↑, Variance ↓ (underfits). Too simple.
PruningCollapsed node → replace with mean/majority of all its leaf predictions. Reduces overfitting.
Reduced Error Pruning — response_new mean of ALL data points in both collapsed branches combined
❓ Handling Missing Values ▾

Introduce a new "Missing" category when the absence itself is informative — i.e., when the reason for missingness is systematically related to the outcome.

Good examples: Emergency operator too busy to record (missingness linked to urgency of event). Doctor skips a measurement because patient is critical (informative missingness).
Not appropriate: Accidental loss (coffee spill, fire) — these are random, not informative.
WEEK 7

Model Evaluation & Ensemble Methods

8 questions
📋 Confusion Matrix Metrics ▾
Key formulas (TP, TN, FP, FN) Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
Accuracy = (TP + TN) / (TP + TN + FP + FN)
F-Score = 2 · Precision · Recall / (Precision + Recall)
Class imbalanceUse F-Score. Accuracy is misleading when one class dominates.
ROC curveY-axis = TPR (Recall), X-axis = FPR = FP/(FP+TN). AUROC = area under ROC curve.
🧪 Experimental Design Considerations ▾
Sampling BiasTraining set not representative of true population distribution.
Order EffectsPresentation order of stimuli influences responses — randomize order.
Floor/CeilingTask too easy (everyone scores 100%) or too hard (everyone scores 0%) — no useful signal.
🪄 Bagging vs Stacking ▾
BaggingStacking
Training data per modelBootstrap sample (subset)Full dataset
CombinationUniform vote/averageLearned meta-classifier
ClassifiersSame algorithmDifferent algorithms allowed
Limited data → prefer❌✅ (uses all data)
Bagging reduces variance — works best when classifiers are uncorrelated. If all classifiers are fully correlated → bagging gives no benefit.
Committee Machines = special case of 2-layer stacking with uniform weights. Stacking is strictly more powerful (learned weights).
WEEK 8

Random Forests, Boosting & Bayesian Networks

9 questions
🌲 Random Forests ▾

Bagging applied to decision trees + random feature selection at each split. Parameter m = number of features considered at each split (m << p).

Decrease m↓ correlation between trees (more diverse). Individual tree quality ↓ slightly.
Increase m↑ individual tree quality. ↑ correlation between trees (less diverse ensemble).
💪 Boosting (AdaBoost) ▾
IdeaTrain classifiers sequentially. Each focuses on mistakes of the previous one.
MisclassifiedWeights of misclassified points are increased. Correctly classified weights are decreased.
Parallel?No — sequential. Bagging CAN be parallel.
Learner typeUses weak learners (e.g. shallow trees / stumps). Bagging can use strong learners.
Noise sensitivityBoosting is MORE sensitive to noise than bagging.
Boosting and bagging both work for regression AND classification.
🕸️ Bayesian Networks — Key Properties ▾
StructureDirected Acyclic Graph (DAG). Edges are directed, no cycles.
MeaningEncodes conditional independence. Does NOT imply causation.
D-separationTool to read conditional independences from graph structure.
Bayes' Theorem P(A|B) = P(B|A) · P(A) / P(B)
Joint probability in BN: P(x₁,...,xₙ) = Π P(xᵢ | parents(xᵢ)). Multiply each node's CPD given its parents.
WEEK 9

Bayesian & Markov Networks

10 questions
🔗 Conditional Independence in Bayesian Networks ▾

A node is conditionally independent of all non-descendants given its parents.

d-separationPath between X and Y is blocked given evidence Z if any node on the path satisfies the blocking condition.
Chain: A→B→CA ⊥ C | B (B blocks the path)
Fork: A←B→CA ⊥ C | B (B blocks)
Collider: A→B←CA ⊥ C (path blocked at B). A NOT ⊥ C | B (observing B opens path!)
Exam strategy: Identify path type (chain/fork/collider) and apply blocking rules to verify independence claims.
🔶 Markov Networks (Undirected Graphical Models) ▾
Markov BlanketIn a Markov network = set of direct neighbours only.
IndependenceA node is independent of all other nodes given its Markov blanket.
FactorizationP(x) = (1/Z) Π ψ_c(x_c) — product of clique potentials divided by partition function Z.
Markov Blanket of node X = its immediate neighbours in the graph. Nodes outside the blanket have NO effect on X given the blanket.
🧮 Computing Joint Probabilities in BN ▾
Chain rule of BN P(all vars) = Π_i P(xᵢ | pa(xᵢ))

To find P(i=1, d=0, g=2, s=1, l=1): multiply P(i=1) · P(d=0) · P(g=2|i=1,d=0) · P(s=1|i=1) · P(l=1|g=2).

To marginalize over a variable (e.g., difficulty d): sum the joint over all values of d.

WEEK 10

Clustering Algorithms

6 questions
🔵 Hierarchical Clustering ▾
LinkageDistance between clustersShape bias
Single LinkMin distance of any pairChain-like / elongated clusters
Complete LinkMax distance of any pairCompact, globular clusters
Average LinkAverage of all pairsCompromise
Single link is best for non-convex / oddly shaped clusters. For compact round clusters → complete link.
🗃️ BIRCH Algorithm ▾

Cluster Feature (CF) of a cluster = (N, SUM, SS): count, vector sum, sum of squares of all points.

Centroid of combined cluster A ∪ B centroid = (SUM_A + SUM_B) / (N_A + N_B)
CFs are additive — can merge two CFs without revisiting original data.
🔎 DBSCAN ▾

Density-based clustering. Two parameters: Eps (neighbourhood radius) and MinPts (minimum points to be a core point).

MinPts ↑ (same Eps)Fewer core points → clusters shrink → more points classified as noise → more clusters (smaller).
ShapesCan find arbitrary shapes — no assumption. Better than k-means for non-convex clusters.
🌀 CURE Algorithm ▾

Uses representative scatter points (not just centroid) to represent cluster shape. Shrinks them toward the centroid by a fraction α.

Shape assumption: None. CURE handles arbitrary cluster shapes, unlike k-means (spherical) or Gaussian EM (elliptical).
WEEK 11

MLE, Gaussian Mixtures & EM Algorithm

5 questions
📐 MLE for Common Distributions ▾
Normal — MLE of μ μ_MLE = (1/n) Σ xᵢ (sample mean)
Geometric Pr(X=k) = (1−p)^(k−1)·p — MLE of p p_MLE = n / Σᵢ xᵢ (reciprocal of sample mean)
Binomial f(x,n,p) = C(n,x)·pˣ·(1−p)^(n−x) — MLE of p p_MLE = Σxᵢ / (n·N) where N = number of samples
Method: Take log-likelihood, differentiate with respect to parameter, set to zero and solve.
🎲 Gaussian Distribution Parameters ▾

A p-dimensional Gaussian needs:

Mean μp independent parameters (one per dimension).
Covariance Σp(p+1)/2 independent parameters (symmetric matrix — upper triangle + diagonal).
Full covariance: p×p matrix, symmetric → p(p+1)/2 unique entries (not p²).
🔄 EM Algorithm & Gaussian Mixture Models ▾

GMM models data as a weighted sum of Gaussian components. Mixing coefficients πᵢ must satisfy: 0 ≤ πᵢ ≤ 1 and Σπᵢ = 1.

Invalid constraint: −1 ≤ πᵢ ≤ 1 is too loose (allows negative probabilities).
EM — two steps per iteration E-step: Q(θ, θᵐ⁻¹) = E_{Z|X,θᵐ⁻¹}[log P(X,Z|θ)]
M-step: θᵐ = argmax_θ Q(θ, θᵐ⁻¹)
E-stepCompute expected complete log-likelihood using current parameters θᵐ⁻¹.
M-stepMaximize Q w.r.t θ to get new parameters θᵐ.
WEEK 12

Reinforcement Learning & Consolidation

7 questions
🤖 RL Framework — State, Action, Reward ▾
State (S)Complete description of environment needed to make decision. Must satisfy Markov property — fully describes what's needed, no memory of history required.
Action (A)Things the agent can do to change the state. e.g. pressing pedals, turning wheel.
Reward (R)Scalar feedback signal. Agent maximizes cumulative reward.
Policy (π)Mapping from states to actions. Goal = find optimal policy.
Self-driving car: State = current position/speed/pedals (observable now). Actions = turn wheel, press pedals (direct actuators). Speed, not acceleration — speed is directly observable.
🎯 Markov Property & Valid States ▾

A state is Markov if it captures all information needed to predict the future — the future is conditionally independent of the past given the present state.

Chess✅ All piece positions on board = full state. ❌ Only your pieces = not enough.
Tic-Tac-Toe✅ Full board snapshot captures all needed info.
Tennis ball position❌ Need velocity too (position alone not Markov).
🔍 Exploration vs Exploitation & Reward Shaping ▾
ExplorationTry new actions to discover potentially better strategies.
ExploitationUse currently known best action.
Non-stationaryIf reward distribution changes over time → must continue exploring. Never stop.
Reward shaping pitfall: Adding intermediate rewards (e.g. +3 for near-hole) can cause the agent to exploit the sub-goal rather than the true goal. Often hinders learning.
Hoeffding's inequality — sample size for estimation n ≈ 200 · ln(20) for 10% accuracy at 90% confidence (coin toss)
